Statistical Test Index

Fukui-first tourism text analysis - public-facing test map

This page collects the statistical tests currently implemented in the repository and links each referenced aggregate output. It excludes raw row-level text, author fields, source URLs, review IDs, and other local data-bearing artifacts.

tracked aggregate outputs reproducible commands descriptive where source/platform differs no raw cross-tool score comparison

Interpretation Boundary

English-language Google reviews use VADER, Japanese-language Google reviews use oseti, and Chinese-language social rows use SnowNLP. Category-share tests and within-tool score diagnostics are reproducible, but raw VADER, oseti, and SnowNLP means are not common-scale evidence.

JP/EN Google review tests are strongest when they keep platform constant. Current Chinese-language social comparisons use Xiaohongshu rows by default; Douyin is available only through the source-sensitivity path.

sentiment_manifest.json 		Chi-square sentiment category, Mann-Whitney sentiment score, bootstrap mean/median score differences, POI-level and cluster bootstrap sensitivity, rating Welch tests, Welch ANOVA, Spearman rating validation, and explicit skipped raw-score parametric tests. Broad aggregate statistical evidence created by the sentiment analysis pipeline. Score-distribution tests are descriptive sensitivity; VADER and oseti raw scores are not common scale.

H1-H3 JP/EN Hypothesis Tests

These same-platform H1-H3 outputs provide descriptive support. Use the within-POI paired Wilcoxon test as the venue-clustering robustness check.

H1 Sentiment Category

Descriptive support for whether English-language and Japanese-language Fukui Google reviews differ in negative, neutral, and positive sentiment category shares. Includes neutral-band sensitivity.

  • Script: scripts/test_h1_sentiment_category_jp_en.py
  • Command: make hypothesis-h1
  • Output: h1_sentiment_category_jp_en.csv
  • Manifest: h1_sentiment_category_jp_en_manifest.json
  • Tests: chi-square, Cramer's V, Holm adjustment across neutral-band sensitivity rows.

H2 Star Rating

Descriptive support using Google review ratings as a common 1-to-5 scale. This is a companion outcome, not raw text-sentiment equivalence.

  • Script: scripts/test_h2_review_rating_jp_en.py
  • Command: make hypothesis-h2
  • Output: h2_review_rating_jp_en.csv
  • Manifest: h2_review_rating_jp_en_manifest.json
  • Tests: Welch t-test, POI-level Welch sensitivity, Mann-Whitney, rating-distribution chi-square.

H3 Reviewed Evidence

Descriptive support for reviewed keyword evidence prevalence for friction, enjoyment, recommendation, and positive evidence.

  • Script: scripts/test_h3_reviewed_evidence_jp_en.py
  • Command: make hypothesis-h3
  • Output: h3_reviewed_evidence_jp_en.csv
  • Manifest: h3_reviewed_evidence_jp_en_manifest.json
  • Tests: chi-square or Fisher exact, risk difference, Benjamini-Hochberg FDR.

Within-POI Robustness

Paired Wilcoxon robustness check comparing English-language and Japanese-language means inside shared Fukui POIs. Effective N is paired POIs, not review rows.

  • Script: scripts/test_within_poi_paired_jp_en.py
  • Command: make hypothesis-within-poi
  • Output: within_poi_paired_jp_en.csv
  • Manifest: within_poi_paired_jp_en_manifest.json
  • Tests: paired Wilcoxon on POI-level rating means and secondary positive-share means.

Cross-Source Baseline Tests

Cross-source tests are descriptive because Google reviews and current Xiaohongshu rows differ in platform, intent, text length, and date quality. Xiaohongshu is the default Chinese layer; Douyin is optional through the source-sensitivity path. Do not interpret raw sentiment score means across tools.
